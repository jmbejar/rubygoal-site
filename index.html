¿Qué es #rubygoal?

RubyGoal es un juego en el cual podrás ser el/la DT de un equipo de fútbol. Participarás contra otros programadores implementando en Ruby la estrategia que va a utilizar tu equipo.

Este es un partido de ejemplo generado por Rubygoal:

¿Cómo instalarlo?

{% highlight bash %} $ gem install rubygoal {% endhighlight %}

¿Cómo ejecutarlo?

Correr el juego con los CoachDefinition de ejemplo {% highlight bash %} $ rubygoal {% endhighlight %} Correr el juego con tu implementación de CoachDefinition {% highlight bash %} $ rubygoal coach_1.rb {% endhighlight %} Correr el juego con tu implementación de ambos CoachDefinition {% highlight bash %} $ rubygoal coach_1.rb coach_2.rb {% endhighlight %}

A tener en cuenta

El siguiente conjunto de reglas debe ser respetado por los participantes del campeonato.

  1. La implementación del método name debe devolver el mismo nombre SIEMPRE.
    No se aceptarán participantes que hagan una implementación de este método que devuelva distintos valores en cada ejecución
  2. La formación debe siempre contener la cantidad correcta de jugadores
    • 1 del tipo :captain
    • 3 del tipo :fast
    • 6 del tipo :average
    Equipos que en algún flujo de la implementación rompan esta regla, serán descalificados
  3. La implementación de los métodos lineup y name no debe ser costoso en tiempo.
    Ambos métodos van a ser invocados muchas veces por segundo y deben correr con esa restricción de tiempo (en el orden de centésimas de segundo). Pruebe su implementación con alguna de las tácticas de ejemplo para ver si el juego corre fluidamente y poder anticipar problemas de performance.
    Queda a criterio del comité organizador descalificar equipos que no cumplan con ese requisito.
  4. No está permitida la modificación en tiempo de ejecución de ninguna de las clases del juego.
